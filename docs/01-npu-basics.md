# 01. Что такое NPU и чем он отличается от GPU

## Очень коротко

NPU — это специализированный матричный сопроцессор. Он умеет ровно одно: быстро и энергоэффективно перемножать тензоры (обычно в INT8/INT16/FP16). За пределами этого NPU умеет очень мало: ни параллельной графики, ни произвольных шейдеров, ни общей памяти как у CPU.

Если на GPU модель ML — один из многих типов нагрузок, то на NPU модель ML — *единственный* тип нагрузки. Этот фокус позволяет ему быть в 5–20× энергоэффективнее GPU на тех же операциях.

## Как NPU устроен внутри

В нашем случае (Hexagon HTP в Snapdragon) выглядит так:

```
            ┌─────────────────────────────────────┐
            │ Hexagon HTP (Tensor Processor)      │
            │                                     │
            │  ┌─────────────────────────────┐    │
            │  │ Scalar/HVX cores (4–6)      │    │  ← обычный код управления
            │  │ VLIW DSP, 1024-bit vectors  │    │     (тензорные циклы)
            │  └─────────────────────────────┘    │
            │  ┌─────────────────────────────┐    │
            │  │ HMX (Hexagon Matrix Engine) │    │  ← собственно "матричник"
            │  │ INT8 / INT16 GEMM,          │    │     систолический массив
            │  │ ~100 TOPS на v81            │    │     ~64×64 MAC за такт
            │  └─────────────────────────────┘    │
            │  ┌─────────────────────────────┐    │
            │  │ VTCM (Vector TCM)           │    │  ← быстрая on-chip память
            │  │ 8–16 МБ, ~6–8 ТБ/с          │    │     для активаций и весов
            │  └─────────────────────────────┘    │
            └─────────────────────────────────────┘
                    │           │
                ┌───┴───┐   ┌───┴────┐
                │ DDR    │   │ ION    │ ← общая память с CPU/GPU
                │ ~67    │   │ shared │     через DMA
                │ ГБ/с   │   │ buffer │
                └────────┘   └────────┘
```

Главное, что нужно понимать с точки зрения программиста:

1. **VTCM крошечная.** 8 МБ — это всё. Если активации не помещаются, начинаются DMA-перетасовки в DDR и производительность падает в разы. Поэтому модели для NPU дробятся: SD 1.5 разрезают на 3 части (text_encoder, unet, vae_decoder), и каждая прогоняется отдельно.

2. **Квантизация — не опция.** HMX живёт в INT8 (с накоплением в INT32) и в FP16. Если модель в FP32 — её сначала прогонят через PTQ/QAT (Post-/Quantization-Aware Training) и только потом скомпилируют под HTP. Большинство публичных «оптимизированных под NPU» моделей — это уже квантованные веса с откалиброванными `scale/zero_point` для каждого тензора.

3. **Граф фиксирован после компиляции.** В отличие от CUDA, где можно динамически собирать op-граф на каждом шаге, QNN-граф собирается один раз (часто в облаке через AI Hub), сериализуется в `.bin`, и на устройстве уже только `QnnContext_createFromBinary` + `QnnGraph_execute`. Никаких eager-режимов.

## Чем NPU отличается от GPU

| Свойство                | NPU (Hexagon HTP)           | GPU (Adreno 740, RTX 4080S)                        |
|-------------------------|------------------------------|----------------------------------------------------|
| Универсальность         | Только тензорная математика | Графика, compute, ML                               |
| Точность по умолчанию   | INT8 / FP16                  | FP16 / FP32 (FP8 на свежих NV)                     |
| Память                  | 8–16 МБ on-chip + общий DDR  | Десятки МБ кэшей + 16 ГБ GDDR6X (на 4080S)        |
| Энергопотребление       | 1–5 Вт под полной нагрузкой  | 250–320 Вт (4080S)                                 |
| Производительность      | ~30–100 TOPS (INT8)          | ~80 TFLOPS (FP16) на 4080S                         |
| Гибкость графа          | Статический, скомпилирован  | Динамический                                        |
| Driver overhead         | Высокий (compile-time)       | Низкий (driver сделает за runtime)                 |

Главный практический вывод: **NPU нет смысла сравнивать с GPU «в лоб» по FLOPS**. Сравнивать имеет смысл по двум вещам:

- **Latency end-to-end** на конкретной модели (от подачи входа до получения выхода).
- **Энергозатраты на запрос** (Дж/inference) — здесь NPU выигрывает на порядок-два.

## Классы NPU в мобильном мире

| Чип                            | NPU                            | Доступ                                          |
|--------------------------------|--------------------------------|-------------------------------------------------|
| Snapdragon 8 Elite Gen 5 / Gen 3 | Hexagon HTP v81 / v75         | **QNN SDK** (наш путь), TFLite, ExecuTorch     |
| Apple A-series / M-series      | Apple Neural Engine            | CoreML (закрытый — без обходных путей)         |
| Samsung Exynos                 | Samsung NPU                    | Samsung Neural SDK, NNAPI                       |
| Mediatek Dimensity             | APU                            | NeuroPilot SDK, NNAPI                           |
| Google Tensor (Pixel)          | TPU lite                       | TFLite + Google Edge TPU delegate              |
| Huawei Kirin                   | Da Vinci NPU                   | HiAI SDK                                       |

NNAPI (Android Neural Networks API) — это попытка Google сделать единый зонтик для всего этого. Реальность: он deprecated с Android 15 (LiteRT теперь основной путь), и драйверы вендоров до него доходят с большой задержкой. Для серьёзной работы с NPU все вендоры тянут собственные SDK — мы поэтому и берём QNN напрямую.
